Multithreading: process of creating multiple threads
use of it is that cpu utilized properly and make things faster
also it means executing multiple thread simultaneously to achieve concurrent execution

Process is independent running program (eg. java, chrome, zoom)
Thread is lightweight process (inside zoom audio, video and recording are separate threads)

We have two options to create thread:
1. By extending thread class
2. By implementing the runnable interface

---

Relationship between start() and run() method:
1. When we use start() method, a new thread will be created and this method is responsible to call run() method
2. When we use run() in place of start() then no new thread will be created but when u use t.run() no multi threading is applied

---

what will happen if we dont override run method?
If you do not override the run() method when working with threads in Java, the resulting thread will be created and started,
but it will not perform any specific task because the default run() method in the
Thread class or Runnable interface has an empty implementation.
